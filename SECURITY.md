# Security Policy

## Reporting a Vulnerability

If you find a security issue in this template, please avoid opening a public
issue with exploit details.

Use GitHub's private vulnerability reporting when available, or contact the
maintainer through the public profile links with:

- A concise description of the issue.
- Steps to reproduce or validate it.
- The affected files, dependencies, or configuration.
- Any suggested mitigation.

This project is a template, so security fixes usually focus on improving default
configuration and documenting safer setup choices for downstream apps.
